## Purpose
Define the initial steps to validate and authorize work, align stakeholders, and create a lightweight plan.

## When to use
Whenever a new project idea or feature proposal is ready to be explored.

## Goals
- Confirm business need and measurable outcome
- Identify stakeholders & champions
- Define success criteria and initial timeline
- Decide go/no-go for planning

## Minimum Deliverables
- Project One-pager (Problem, Goal, Success Metrics)
- Stakeholder list & communication plan
  - Include cross-functional stakeholders (Operations, Security, UX, Data, Support)
- High-level timeline and key milestones
- Initial risk list
  - **Security Lead**: Security and compliance risks
  - **Operations Manager**: Infrastructure and operational risks
- Resource needs (team roles, rough effort estimate)
  - Consider which cross-functional roles are needed (see [octoacme-role-onboarding-checklist.md](octoacme-role-onboarding-checklist.md))

## Project One-pager Template
- Project name:
- Problem statement:
- Objective / Goal (SMART):
- Success metrics:
  - **Data Analyst**: Ensures metrics are measurable and trackable
- Primary stakeholders:
  - Include cross-functional roles as appropriate
- Suggested timeline / milestones:
- Quick risks & dependencies:
  - Security considerations (Security Lead)
  - Infrastructure requirements (Operations Manager)
  - User experience concerns (UX/UI Designer)
  - Support and documentation needs (Support Lead)
- Proposed team / roles:
  - Core team: PM, PdM, Developers
  - Cross-functional: Operations Manager, UX/UI Designer, Security Lead, Data Analyst, Support Lead (as needed)
  - See [octoacme-roles-and-personas.md](octoacme-roles-and-personas.md) for role definitions

## Initiation Checklist
- [ ] One-pager completed and reviewed by Product Lead
- [ ] Sponsor / Stakeholder alignment (email or meeting)
- [ ] Decision: Approve to move into planning?
- [ ] Create repo or project board skeleton
- [ ] Add initial artifacts to repo (docs/ or .copilot/)
- [ ] Identify and assign core roles (PM, PdM, Tech Lead)
- [ ] Determine which cross-functional roles are needed
- [ ] Review role onboarding checklist for selected roles (see [octoacme-role-onboarding-checklist.md](octoacme-role-onboarding-checklist.md))

## Decision Gate
Move to planning when:
- Success metrics are clear
- Stakeholders agree on priority
- Team availability is confirmed
